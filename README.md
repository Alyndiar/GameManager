# Game Backup Manager

Windows desktop application for scanning game backup roots, normalizing names, moving archives, and assigning folder icons with an image-processing pipeline.

## Environment

- OS: Windows (uses Explorer integration, `desktop.ini`, `attrib`, PowerShell elevation, and Windows Credential Manager)
- Python: 3.11 or newer
- Core Python dependencies (from `requirements.txt`):
  - `PySide6`
  - `psutil`
  - `Pillow`
  - `pillow-avif-plugin`
  - `requests`
  - `rembg`

### Optional runtime components

- TeraCopy (optional): used as a selectable move backend for compatible move operations.
- Advanced text extraction and GPU acceleration are capability-based:
  - PaddleOCR path requires `paddleocr` (and optional CUDA/cuDNN runtime availability)
  - OpenCV DB path requires `opencv-python` and `numpy`
  - Torch/ONNX runtime availability influences GPU status and cutout acceleration
- OpenCV DB model file is required for OpenCV DB text extraction:
  - Set `GAMEMANAGER_OPENCV_DB_MODEL`, or
  - Place `DB_TD500_resnet18.onnx` in `.gamemanager_data/models/opencv_db/`

## Installation

```powershell
python -m pip install -r requirements-dev.txt
```

## Run

```powershell
python -m gamemanager
```

Standalone icon tool:

```powershell
python -m iconmaker_gui
```

`GameManager` and `IconMaker` are mutually exclusive at runtime (only one can run at a time).

## Test

```powershell
.\tools\run_tests.ps1
```

This always runs tests in the `GameManager` conda environment (`conda run -n GameManager`),
independent of your currently active shell environment.

## Data, Cache, and Secret Storage

- Default app data root: `.gamemanager_data` (project-local).
- Override data root with:
  - `GAMEMANAGER_DATA_DIR=<path>`
- Persistent project-local files include:
  - SQLite DB: `.gamemanager_data/manager.db`
  - Caches: `.gamemanager_data/cache/`
  - Temp worker files: `.gamemanager_data/tmp/`
  - Crash logs: `.gamemanager_data/logs/crash.log`
  - Model/cache directories under `.gamemanager_data/models/`
  - Web capture session staging: `.gamemanager_data/web_capture_session/`
- Legacy `%APPDATA%\GameBackupManager` DB/cache is auto-migrated into project-local storage when present.
- API keys are stored in Windows Credential Manager (target names prefixed with `GameManager:`), not plain-text project files.

## Features

### 1) Root scanning and inventory view

- Add/remove root folders.
- Scans root-level entries (folders and files) per configured root.
- Computes and displays:
  - full name
  - cleaned name
  - modified/created timestamps
  - size
  - source/root label
- Root panel supports sorting by source label, drive name, or free space.
- Root label display modes: letter/mountpoint, drive name, or both.
- Right panel supports list and icon views, sortable columns, and icon-size scaling.
- Global type-to-filter by cleaned name.
- Duplicate-only toggle to show only entries sharing the same cleaned name.

### 2) Name normalization and tagging workflow

- Cleaned names are generated by:
  - separator normalization (`_`/`.` to spaces where appropriate)
  - approved suffix-tag removal
  - trailing version/build token stripping
  - whitespace normalization
- Tag Finder scans suffix tag candidates from current inventory.
- Tag review dialog classifies candidates as:
  - `approved`
  - `non_tag`
  - `ignore`
- Decisions are persisted and used on future scans.

### 3) Cleanup (safe rename) workflow

- Builds rename plan for root-level items.
- Preview marks each item as `ready`, `unchanged`, or `conflict`.
- Apply operation executes only selected safe renames.
- Conflicts remain manual.
- Progress, cancellation, and result report included.

### 4) Archive move workflow

- Targets root-level archive/media files with extensions:
  - `.iso`, `.zip`, `.rar`, `.7z`
- Builds destination as `<root>\<cleaned_name>\<original_filename>`.
- Conflict actions:
  - `skip`
  - `overwrite`
  - `rename` (manual filename)
  - `delete_destination`
- Supports two backends:
  - System move
  - TeraCopy (auto-detect or manually locate executable)
- Includes preview dialog, progress, cancellation, and summary report.

### 5) Drag/drop move between roots

- Drag selected entries from right panel onto a root in the left panel.
- Validates destination free space and destination conflicts before move.
- Uses selected move backend (System/TeraCopy) when possible.

### 6) Open, manual rename, and delete workflows

- Open selected entry in Explorer (`Ctrl+O`).
- Manual rename for a single selected entry with validation and confirmation.
- Delete selected entries with:
  - grouped duplicate-aware review dialog
  - interactive progress and cancellation
  - elevation fallback via `RunAs` subprocess if direct delete fails

### 7) Folder icon assignment workflow

- Assign icons to selected folders.
- Candidate source providers:
  - SteamGridDB
  - Iconfinder
  - Steam fallback assets (when applicable)
- SteamGridDB resource controls:
  - resource type enable/disable (`icons`, `logos`, `grids`, `heroes`)
  - drag-drop priority ordering
- Supports:
  - candidate selection
  - local image selection
  - external web capture workflow (captures new downloads from `E:\Downloads`)
- Optional InfoTip text written to `desktop.ini`.
- Icon generation pipeline includes:
  - template style selection (built-in + `IconTemplates/*.png`)
  - background cutout engine selection
  - text extraction/preservation methods
  - border shader tinting
  - multi-size ICO generation
- Icon apply is isolated in a subprocess to avoid crashing the UI process.
- Applies folder icon by writing local `.ico` + `desktop.ini`, setting folder/system attributes, and refreshing shell.

### 8) Icon framing dialog (advanced)

- Full compositing preview canvas with pan/zoom.
- Template overlay controls.
- Background removal controls (thresholds, erosion, feather, post-process).
- Text extraction controls:
  - methods: Disabled, ROI Guided, PaddleOCR, OpenCV DB, Heuristic
  - strength/feather/group settings
  - outline/shadow/glow controls
  - ROI draw/clear
  - seed color picking
  - manual add/remove text marks with undo/redo
  - debug alpha-mask preview
- Layer visibility toggles: background/template/cutout/text.
- Exports a final composited PNG used directly by icon assignment/conversion.
- Shared with standalone `IconMaker` to keep icon construction parity between both GUIs.

### 9) Icon converter tool

- Standalone "Image to Icon Converter" dialog.
- Converts local images to multi-size `.ico`.
- Reuses the same template/cutout/text/border pipeline as folder icon assignment.
- Supports framing-first flow and direct conversion flow.

### 10) Template tools

- Template Batch Prep:
  - drag-drop files/folders
  - normalize/crop images to square PNG templates
  - outputs sequential `templateNNN.png` files in `IconTemplates/`
  - live checkerboard preview and per-item progress
- Template Transparency:
  - open existing template PNGs
  - remove black/near-black backgrounds with flood-fill strategy
  - live preview and in-place apply/cancel

### 11) Icon path repair

- Scans root folders for `desktop.ini` entries using absolute/external icon paths.
- Moves icon files into their owning folder.
- Rewrites `IconResource` to local relative path.
- Repairs attributes and refreshes shell.

### 12) Performance and runtime controls

- Performance settings include:
  - folder-size worker count
  - progress update interval
  - directory-size cache enable/disable
  - directory-size cache max entries
  - startup model preload mode (`off`, `minimal`, `full`)
- GPU status indicator shows runtime availability for:
  - selected cutout engine
  - PaddleOCR
  - OpenCV DB
  - Torch
- Startup prewarm runs in a subprocess and reports progress in the status bar.
- Global cancel button supports refresh, background ops, interactive ops, and TeraCopy sessions.

## Batch CLI Utilities

### 1) Batch image prep to normalized PNG

```powershell
python -m gamemanager.tools.image_prep_batch `
  --input "E:\Downloads\Icons" `
  --output-dir ".\PreparedIcons" `
  --recursive `
  --size 512 `
  --padding-ratio 0.00 `
  --min-padding-px 1
```

### 2) Batch template transparency conversion

```powershell
python -m gamemanager.tools.template_transparency_batch `
  --input ".\IconTemplates" `
  --output ".\IconTemplatesProcessed" `
  --recursive `
  --threshold 22 `
  --mode max
```
